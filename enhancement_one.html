Overview

View Project in Repository

The Inventory Management App is a mobile application that allows users to create an invenotry of their choosing, and perform various functions such as adding and removing items, as well as updating them. The application also sends users push notifications when an items inventory quantity is low. Developed originally in CS-360: Mobile Architect & Programming, the app underwent significant upgrades in Enhancement One as part of my CS-499: Computer Science Capstone.

These enhancements focused on implementing industry-standard solutions, improving user interactivity, and ensuring secure, reliable data handling. The applications improvements include creating a way to allow users to stay signed in as well as log out, create a post notficiations system, and lastly add proper input validation to the system.

Key Improvements

  • Input Validation: Ensured that there are default values for the name, sku, and amount to prevent the application from crashing. Also made it so the system checks that the value passed in is valid before it runs the database command(s).
  • Improved SMS Notification system: Fixed SMS notification system to only send the user a text when an item is deleted.
  • Created Post Notification system: Added a notifcation system that sends the user an app notification when an item is low in stock.
  • Login and Logout Functions: Added shared preferences to ensure user stays signed in even if they close the application and re-open it.

Technical Implementation

The following changes were implemented to enhance the app:

  • Created the InputVlaidation class to validate inputs. Furthermore set the default values of name= "No Name", sku="N/A", and amount "1" if the values passed in are not valid.
  • Integrated the POST_NOTIFICATIONS permission to allow the system to send app notifcations, while also ensuring that the SMS notifications were still active. This allows the app to send an app notification when something is low in stock, and a text message when an item is deleted.
  • Enhanced the Login/Sign Up process to keep users signed in through the use of shared preferences.
  • Implemented detailed user feedback mechanisms, including error messages and success notifications, to enhance usability.
